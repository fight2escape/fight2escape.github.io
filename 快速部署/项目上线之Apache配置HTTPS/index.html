    将服务器升级到CentOS7.2后，遇到问题时都比较尴尬，所幸我有足够的耐心不断试错，最后也得以修成正果。

    实现方法

    之前写的不少过分详细的博文，可能有点侮辱大家的智商了，后面尽量挑我自己实践总结的百度不到的东西讲。

    • 证书申请、下载、上传
      • 首先是证书的申请，腾讯云为我们提供了免费的一键获取渠道。
      • 下载证书，解压，找到相应web服务器如Apache文件夹，上传到服务器指定位置，如 /usr/local/apache/conf/ ,如果没有则创建相应目录
    • Apache的mod_ssl模块安装
      • 安装Apache SSL支持模块:# yum install -y mod_ssl （默认yum安装httpd是没有安装该模块的，安装后自动生产/etc/httpd/conf.d/ssl.conf文件）并生成证书
    • https虚拟主机配置
      • 之所以折腾了很久，主要还是因为我给自己配置二级域名的虚拟主机，导致网上的大多数教程对我都不适用，如果有和我一样的直接进行如下配置即可：
      • 找到虚拟主机配置文件，我的是/etc/httpd/conf.d/vhost.conf，根据之前配置http的方式来配置https。只是把80端口库改成443，然后再链接到HTTPS证书。如： <VirtualHost www.domain.com:443> DocumentRoot "/var/www/html" ServerName www.domain.com SSLEngine on SSLCertificateFile /usr/local/apache/conf/2_www.domain.com_cert.crt SSLCertificateKeyFile /usr/local/apache/conf/3_www.domain.com.key SSLCertificateChainFile /usr/local/apache/conf/1_root_bundle.crt <Directory "/var/www/html"> Options FollowSymLinks AllowOverride All Order allow,deny Allow from all </Directory> </VirtualHost>
      • 这里把域名都改成自己的，特别注意下面的 2_www.domain.com_cert.crt，修改后应该是这样2_blog.fight2escape.club.crt
    • 开放端口
      • 接下来还需要在腾讯云的安全组中开放443端口给HTTPS用
      • 不放心的可以使用命令查看相应的端口开放情况 firewall-cmd --query-port=443/tcp
      • 如果显示no，可以用命令打开，其他端口的话改下数字即可 firewall-cmd --add-port=443/tcp
      • 配置完成后，重新启动 Apache 就可以使用https://www.domain.com来访问了你的网站了

    开小灶

    当然，如果你之前没有配置过虚拟主机的话，那可以参考官方教程，但如果你也是使用CentOS7的话就不行了，可以对照下面的补充做修改，因为官方教程中需要的文件我都没有，当然，除了配置文件这个问题，其他的和上面一样。 - 官方教程

    • 找不到配置文件的处理
      • 编辑Apache配置文件httpd.conf 文件，位置是 /etc/httpd/conf/httpd.conf
      • 找到 ```

      LoadModule ssl_module modules/mod_ssl.so,

      Include conf/extra/httpd-ssl.conf

      ，去掉前面的#号注释，如果没有的话，找到相应的`Include`包含语句，可能是放在了其他地方，比如`conf.modules.d/*.conf`; - 进入相应的目录，找到**00-ssl.conf**，进入编辑，应该已经有了LoadModule，加上我们的Include包含其他文件， - 然后进入我们设置的目录`conf/extra/`，没有则新建目录，接着新建文件httpd-ssl.conf，加入如下内容 DocumentRoot "/var/www/html" ServerName www.domain.com SSLEngine on SSLCertificateFile /usr/local/apache/conf/2_www.domain.com_cert.crt SSLCertificateKeyFile /usr/local/apache/conf/3_www.domain.com.key SSLCertificateChainFile /usr/local/apache/conf/1_root_bundle.crt Options FollowSymLinks AllowOverride All Order allow,deny Allow from all ``` - 注意把域名都改成自己的，还有下面的 2_www.domain.com_cert.crt，修改后应该是这样2_blog.fight2escape.club.crt

    MySQL的下载安装

    数据库(DB)的种类很多，比如网状DB、关系DB、树状DB、面向对象DB，商业应用中主要还是关系型数据库，具体的产品有Oracle、MySQL、DB2、MsSQL等，我们选择其中的MySQL是因为，这个软件是免费的开源的，而且性能也很强劲。接下来请看详细报道

    • 首先下载MySQL 目前5.7，比较大但功能完善
    • 因为是msi文件，比较方便，如果是下载了绿色版的，就得手动配置，具体方法这个总结得不错，小心不再赘述

    MySQL学习之路指北

    接下来就可以开始MySQL的快乐编程了，万事开头难，安装都解决了，还有什么问题不能解决！

    • 首先，你得学会如何开启MySQL的小黑窗，小心建议你现在黑窗上多加练习，还是那句话，打好基础。
    • 然后，你需要学习如何创建、删改和选择数据库
    • 接着，你想在数据库中创建多个表，但定义字段时你得先知道MySQL的数据类型
    • 之后是对表的创建和删改，如果你在下一步中出现乱码，那一定是创建数据库和数据表时没有设置好编码规则，可参看下面的乱码处理
    • 继续，向表中插入数据，在单个表中做查询，更新，删除以及分组排序
    • 难度升级到多个表，在连接多表的情况下做复合的增删改查，能够分组排序去重
    • 到这里，你已经能够满足一般应用软件的需求了，往后就是进阶的用法，如事务、索引、导入导出、正则表达式等。
    • 如果你对数据库感兴趣，还可以学下数据库原理，学习下各种范式，深化对数据库的理解
    • 网上教程很多，这里推荐一份MySQL菜鸟教程

    MySQL数据库乱码

    乱码主要是因为字符集不统一导致，就好像你说英语我说粤语，对彼此来说都是乱码，你需要统一四个地方的字符集：

    • 文件本身编码格式，通过设置相关编辑器的编码格式或保存格式
    • 代码(如php/html)字符集设置，用相应代码明确设定所用编码规则
    • 数据库字符集
    • Apache默认字符设置，通过修改配置文件来设置

    这里推荐UTF-8编码，因为它可以兼容世界上所有语言，而在数据库中，你需要做的是

    • 创建数据库的时候：
      CREATE DATABASE `test` CHARACTER SET 'utf8' COLLATE 'utf8_general_ci';
    • 建表的时候 CREATE TABLE `database_user` ( `ID` varchar(40) NOT NULL default '', `UserID` varchar(40) NOT NULL default '', ) ENGINE=InnoDB DEFAULT CHARSET=utf8;

    CentOS7.2上搭建LAMP环境

    • 主要内容如下：

      • Apache
      • MariaDB
      • php
      • 测试是否搭建成功

    • 字数：900+

    CentOS7新增了很多特性，除了内核更新到3.10外，支持 Linux 容器、Open VMware Tools 及 3D 图像能即装即用，转用 systemd、firewalld 及 GRUB2 ，而默认的文件系统为XFS等等,可以说是比较重大的升级。笔者尝试了使用Centos7搭建LAMP服务器平台，记录如下。

    Apache

    • Centos7默认已经安装httpd服务(Apache)，只是没有启动。如果你需要全新安装，可以使用命令yum install -y httpd（-y表示确定安装）
    • 启动Apache服务：systemctl start httpd,CentOS7后转用systemctl，原来的service还可以使用
    • 设置开机自动启动：systemctl enable httpd

    MariaDB

    • 甲骨文公司收购了MySQL后，有将MySQL闭源的潜在风险，因此社区采用分支的方式来避开这个风险。过去一年中，大型互联网用户以及Linux发行商纷纷抛弃MySQL，转投MariaDB阵营。MariaDB是目前最受关注的MySQL数据库衍生版，也被视为开源数据库MySQL的替代品。在大多数情况下，你完全可以卸载MySQL然后安装MariaDB，然后就可以像之前一样正常的运行。

    • 所以如果你尝试yum install -y mysql来安装mysql的话，是找不到mysql包的。这是你得使用如下命令yum -y install mariadb-server mariadb

    • 安装完成后启动服务：systemctl start mysqld，注意这里 如果mysqld不行就改成mariadb
    • 同样设置开机启动: systemctl enable mysqld
    • 和以前使用mysql一样操作即可

    PHP

    • 安装php主程序和mysql驱动包 yum -y install php php-mysql
    • 安装PHP组件，使PHP支持 MariaDB。yum -y install php-gd php-ldap php-odbc php-pear php-xml php-xmlrpc php-mbstring php-snmp php-soap curl curl-devel
    • 重启Apache服务器： systemctl restart httpd

    测试是否搭建成功

    • 用浏览器访问服务器的地址http://主机IP或域名
    • 如果打不开，可能是因为相应端口未开启。不是云服务器的话，修改防火墙设置即可，方法如下
      • 查询80端口是否打开
        • 命令如下： firewall-cmd --query-port=80/tcp
        • 返回yes 表示打开
      • 打开80端口
        • 命令如下：firewall-cmd --add-port=80/tcp
        • 返回success表示打开成功

    • 如果是云服务器的话，在网页的控制台中一般会有安全组的设置，即使在终端设置了打开端口，也是无效的，需要在相应的安全组中进行设置，开放相应的端口。这里以腾讯云为例：

      • 选择安全组，新建
      • 可以选择模板，把相应的端口号改下，如图
      • 保存后选择相应云服务器去匹配刚才添加的安全组，如果是默认开放全部端口的顺便改下，linux默认开放22号用于SSH连接。
      • 保存后再次访问服务器地址，如果还不行，尝试将两种方法都设为开启即可

    • 再测试下PHP

      • 在站点根目录下新建一个文本vim /var/www/html/info.php
      • 按i进入编辑模式,写入代码<?php phpinfo(); ?>
      • 按左上角Esc键，输入:wq回车保存退出。
      • 在浏览器中通过访问http://主机IP或域名/info.php，如果显示了PHP相关信息则正确

    • 至此，LAMP环境搭建完成。

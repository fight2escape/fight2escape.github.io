    接下来开始环境的搭建，看着很多，憋怕，只是做个选择题

    进入官网

    1. 打开官网

    2. 小心直接给你windows平台的下载页面

    版本号

    1. 如你所见，7.1/7.0/5.6，选哪个版本好呢？
      • 你说6.x的去哪了，哦，你看5.6是不是带个6，好巧啊。
      • 7.x性能大幅提升，语法上稍有微调，升级平滑可谓相当给力。
      • 但是刚入门还是推荐5.6，虽然老了脚步不够快了，但是成熟稳重啊，什么大风大浪没见过，有个小问题分分钟解决了。

    32位和64位

    1. 细心的你又发现5.6的VC11后面不一样，有X86和x64。
      • 嗯，没错，x64指的是操作系统是64位的，而x86指32位的，下载对应版本的即可。
      • 你问32位的为啥叫x86?
      • 简单科普下，32位64位指的是cpu内寄存器的地址位数，而以前很多是用8086类处理器(cpu)，比如286/486/i386/i686，以86为后缀，x占位，x86就表示cpu中的一种架构，后来口口相传干脆就叫32位，so，but，学计算机还是不能不了解。
      • 你不知道自己的电脑是几位的？
      • 快捷键打开瞅瞅：win（空格左边）+Pause（F12右边）
      • 你说你的是32位操作系统,基于X64的处理器？
      • 莫方，那就是你的电脑cpu是64位的（这个叼），不过你安装的操作系统是32位的，因为是在操作系统之上执行，所以得按操作系统的位数来，32位不能安装64位的软件。

    线程安全与否

    1. 继续，你说后面还分Non Thread Safe和 Thread Safe，怎么选？
      • 这个简单记住，一般用Apache服务器的就用 Thread Safe，用微软IIS的就用Non Thread Safe

    开发者or使用者

    1. 憋跑，再做个选择就好了。
      • 至于最后选择Zip还是Debug Pack
      • Debug Pack明显小得多，可惜不是你。
      • 作为phper，作为使用者，我们用的是Zip。
      • Debug Pack是PHP的底层，给底层开发人员用的。

    根据以上分析对号入座下载好后，解压后配置环境变量即可

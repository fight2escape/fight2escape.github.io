    快速上手docker

    docker是什么

    团队开发中，需要保证开发、运维环境的一致性，开发和运维团队以及新员工的加入，都会导致沟通成本的增加，导致协作效率低下，产品交付延迟。 Docker的出现正好解决了这一痛点，将不同发行版之间的差异对用户透明化，并能保证软件版本和编译参数的一致性。用户可以利用docker来打包自己的应用，使应用在不同的团队中共享，并进行快速的部署移植。

    docker运用场景

    • 作为云主机使用

      • 相比虚拟机来说，容器使用的是一系列非常轻量级的虚拟化技术，使得其启动、部署、升级跟管理进程一样迅速。

    • 作为服务使用 > 只要把需要运维的各种服务打包成标准的集装箱，就可以在任何能运行Docker的环境下跑起来

      • Web应用服务
        • 因为web应用程序一般是无状态的，随着业务访问量增减，用同样的镜像新建、销毁容器即可轻松实现伸缩
      • 持续集成和持续部署
        • 从代码编写完成推送到git/svn后，自动触发后端CaaS平台将代码下载、编译并构建成测试Docker镜像，再替换测试环境容器服务，自动运行单元/集成测试，最后测试通过后，马上就能自动将新版本镜像更新到线上，完成服务升级。
      • 微服务架构使用
        • 每个容器里可以使用完全不同环境的镜像服务，容器启动即产生了一个独立的微服务主机节点（独立的网络IP），上层服务与下层服务之间服务发现通过环境变量注入、配置文件挂载等多种方式灵活解决，而且还可以直接将云平台提供的各种云服务与自定义的微服务整合组成一个强大的服务集群。
      • 跑一次性/定时任务
        • 制作好一次性程序运行的Docker镜像，当需要运行的时候实时通过镜像创建出容器来执行任务，程序执行完成容器自动退出释放资源。

    Docker搭建(CentOS7.2)

    编写构建开发环境的Dockerfile文件，记录环境构建过程中碰到的问题与解决方法

    环境搭建的过程

    • yum -y install docker
    • systemctl start docker
    • docker version
    • docker run hello-world

    Dockerfile

    # nginx
# 基础镜像
FROM centos:latest

# 维护者信息
MAINTAINER fight2escape/fight2escape@gmail.com

# 镜像操作命令

RUN yum -y install wget  #安装wget包，用于获取安装软件包
RUN yum install yum-priorities -y 
RUN cd /usr/local/src  
RUN wget http://nginx.org/packages/centos/7/noarch/RPMS/nginx-release-centos-7-0.el7.ngx.noarch.rpm  
RUN rpm -ivh nginx-release-centos-7-0.el7.ngx.noarch.rpm  
RUN yum install -y nginx  
RUN systemctl enable nginx.service     #设置开机启动

# 容器启动命令

EXPOSE 80 443
CMD ["nginx","-g","daemon off;"]

    遇到的问题与解决方法

    • yum -y install docker，遇到Error downloading packages，提示xxx:Cannot allocate memory

      • 由于系统内存不足造成，简单地先重启了服务器后没有再报错

    • docker run -d -P training/webapp python app.py，运行测试demo，通过浏览器访问失败

      • 经过排查，发现是因为该容器所用端口未设置开放
      • docker ps查看相关容器所用端口
      • 在云服务器的网页控制台里，添加相关端口到安全组中
      • firewall-cmd --add-port=32679/tcp利用Firewalls来打开相关端口（如果没有firewall则安装后启动该服务）
      • 之后可以通过firewall-cmd --remove-port=32679/tcp来关闭测试端口

    • 如何进入运行中的容器

      • 1.3版本后提供了一个新的命令exec
      • 如：docker exec -it 容器ID或名称 /bin/bash

    • docker pull 镜像 速度太慢

      • 使用DaoCloud提供的加速器
      • curl -sSL https://get.daocloud.io/daotools/set_mirror.sh | sh -s http://31336890.m.daocloud.io

    • 启用了加速器后重启失败

      • systemctl status docker -l查看详细报错
      • unable to configure the Docker daemon with file /etc/docker/daemon.json: invalid character '}' looking for beginning of object key string\n"
      • vim /etc/docker/daemon.json，原来是json对象的最后多了个逗号，去掉即可解决

    • 在容器中启动服务时报错Failed to get D-Bus connection: Operation not permitted

      • 报这个错的原因是dbus-daemon没能启，systemctl并不是不能使用。
      • 将CMD或者entrypoint设置为/usr/sbin/init即可，docker容器会自动将dbus等服务启动起来。如下：
      • docker run --privileged -ti --name test docker.io/centos:7 /usr/sbin/init

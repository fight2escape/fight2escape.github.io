    快速上手php扩展

    php扩展

    扩展库是php扩展的功能，比如php本来不支持操作某种功能，但在新版本想对它提供支持，就以扩展的方式来提供，这样我们在配置php时，如果我们不用此功能，我们就可以让php不加载他，从而节省服务器资源，提高其性能

    如果把php比作一个人的话，各种可以让人使用的各种工具就好比php扩展，可以按需取用，针对不同场景不同要求需要用到不同工具时，可以任意取用，而不必嵌入到人的肉体上永远带着。

    pecl和pear

    这么好的东西已经有了官方版本，分别是PEAR和PECL，我们可以直接利用别人造好的轮子快速实现业务需求。当然，如果你已经足够优秀，也可以造一个轮子来让别人使用

    • PEAR 是“PHP Extension and Application Repository”的缩写，即PHP扩展和应用仓库。
    • PEAR 将PHP程序开发过程中常用的功能编写成类库，涵盖了页面呈现、数据库访问、文件操作、数据结构、缓存操作、网络协议、WebService 等许多方面，用户可以通过下载这些类库并适当的作一些定制以实现自己需要的功能。避免重复发明“车轮”。PEAR 的出现大大提高了PHP 程序的开发效率和开发质量。
    • PECL 是“PHP Extension Community Library”的缩写，即PHP 扩展库。PECL 可以看作PEAR 的一个组成部分，提供了与PEAR 类似的功能。

    • 不同的是PEAR的所有扩展都是用纯粹的PHP代码编写的，用户在下载到PEAR 扩展以后可以直接使用将扩展的代码包含到自己的PHP 文件中使用。而PECL是使用C 语言开发的，通常用于补充一些用PHP难以完成的底层功能，往往需要重新编译或者在配置文件中设置后才能在用户自己的代码中使用。

    • 这里是PECL官网和PEAR官网，可以在这里搜索下载所需安装包进行安装

    查看已安装扩展

    首先我们可以先看下目前已经安装的扩展，主要有以下几种方法

    • phpinfo()

      • 输出 PHP 当前状态的大量信息，包含了 PHP 编译选项、启用的扩展、PHP 版本、服务器信息和环境变量（如果编译为一个模块的话）、PHP环境变量、操作系统版本信息、path 变量、配置选项的本地值和主值、HTTP 头和PHP授权信息(License)。
      • 编写文件<?php phpinfo(); ?>
      • ctrl+F 查找各种扩展，如果找不到就是没有安装或开启

    • array get_loaded_extensions ([ bool $zend_extensions = false ] )

      • 返回所有编译并加载模块名的 array
      • 参数为true的话只返回 Zend 扩展，并非类似 mysqli 的普通扩展。默认是 FALSE (返回普通扩展)。
      • 编写文件<?php print_r(get_loaded_extensions()); ?>

    • bool extension_loaded ( string $name )

      • 检查一个扩展是否已经加载
      • 扩展名称，大小写不敏感。
      • 如果 name 指定的扩展已加载，返回TRUE，否则返回 FALSE。
      • 编写文件<?php echo extension_loaded('gd'); ?>

    • php -m

      • 在php-cli，即命令行中可以使用php -m命令来查看

    管理扩展

    php扩展是个好东西，但也不要滥用，按需开启节省服务器资源

    • 管理扩展主要通过配置php.ini文件
      • windows下一般在安装目录下的一级目录里
      • linux下可以通过命令php --ini进行查看
        • Loaded Configuration File: /usr/local/php/etc/php.ini
    • 查看 extension_dir
      • 这是扩展安装所在的文件夹，一般是同级下的ext
      • 扩展编译后生成.so文件，放一起是方便统一管理
      • 接着在php.ini文件中进行配置即可
    • 在php.ini中找到类似如下的语句;extension=pdo.dll或;extension=pdo.so
      • 这句话的意思是，添加pdo扩展，前面的分号表示注释，去掉分号即可实现添加
      • .dll和.so都是动态链接库，windows下是.dll，linux下是.so
        • 动态链接库的意思是，你可以将各种函数方法等进行封装，然后编译成动态链接库，这样以后只要引用这个.dll就能实现其中提供的方法
        • 好处是，你可以用c语言等来实现这个库供php等其他语言调用，提高执行效率

    直接安装（以pecl为例）

    如果我们需要的扩展在php.ini中不存在，那就需要我们自己手动添加了，这里说的直接安装，意思是扩展已经编译好了，我们只需要下载后放到指定位置并做配置就行，这里仅提供思路

    windows下

    • 可以在PECL官网中搜索下载对应扩展，如果你下载的是源码包，那么就需要后面的“编译安装”了
    • 这里要注意下载的版本必须正确，具体要参考一下几个参数：php版本，vc版本（扩展是基于c/c++的运行环境的），nts/ts（非线程安全/线程安全），x86/x64（操作系统位数）
    • 如果不清楚，可以在phpinfo()中查看PHP Extension Builder
    • 把下载好的.dll文件放到上文中的extension_dir所在目录中，默认是和php.ini同级的ext目录
    • 然后在php.ini中添加一句extension=.dll，是你下载的扩展名
    • 别忘了重启web服务器（如apache、nginx、IIS）

    linux下

    • 同理，下载解压缩后，将其中的.so文件放到扩展所在目录，然后在php.ini中添加配置后重启相关服务即可

    编译安装php时同时开启

    php源码安装包中有个ext目录，预先存放了一些常用的扩展源码，我们可以在编译的同时也对这些扩展进行编译安装，限于篇幅下面仅作简单介绍，比如像这样配置参数:

    • 其中-with 或-enable表示开启，-disable表示关闭
    • ./configure --prefix=/usr/local/php -with-config-file-path=/usr/local/php/etc -with-config-file-scan-dir=/usr/local/php/etc/php.d -with-apxs2=/usr/local/apache/bin/apxs -disable-fileinfo -with-mysql=mysqlnd -with-mysqli=mysqlnd -enable-pdo -with-pdo-mysql=mysqlnd -with-iconv-dir=/usr/local -with-freetype-dir -with-jpeg-dir -with-png-dir -with-zlib -with-libxml-dir=/usr -enable-xml -disable-rpath -enable-bcmath -enable-shmop -enable-exif -enable-sysvsem -enable-inline-optimization -with-curl=/usr/local -enable-mbregex -enable-mbstring -with-mcrypt -with-gd -enable-gd-native-ttf -with-openssl -with-mhash -enable-pcntl -enable-sockets -with-xmlrpc -enable-ftp -enable-intl -with-xsl -with-gettext -enable-zip -enable-soap -disable-debug

    后编译安装

    如果你下载的扩展包中没有.so文件，那这就是需要我们自己手动编译的源码包了。

    • 我们在一开始安装php时很难把我们需要用到的扩展同时也编译好，在我们编译了php之后需要另外加其他扩展的话，我们就暂且称之为后编译安装
    • 后编译安装的过程类似，下面先介绍一般思路和方法，以linux为例
      • 首先是下载并解压相应的源码包
      • 进入解压后的文件夹，执行phpize生成configure可执行文件
        • configure用于进行编译扩展前的配置
        • phpize 是属于 php-devel 中的东西，没有的话需要根据系统进行安装，主要是设定 php 外挂模块的一些设定
      • 执行./configure --with-php-config=/usr/local/php/bin/php-config进行环境检查
        • 加上--with-php-config则编译完成之后，将会自动把生成的.so放到了默认的php扩展目录下，而不用我们自己手动拷贝。
        • 注意这里的路径仅供参考，php-config配置文件一般在你安装目录下的/bin目录下。
        • 安装具体的扩展时，可能还需要配置其他具体参数
      • make && make install
        • make命令是进行编译
        • make install 是将编译好后的文件移动到相应目录下，此命令可能需要root权限
        • 执行完后，成功的话会给出扩展所在目录，如/usr/local/php/lib/php/extensions/no-debug-non-zts-20100525/，可以进入该目录查看具体的扩展名，供下面的配置使用
      • 配置php.ini，可以直接在文件最后添加一句extension=/usr/local/php/lib/php/extensions/no-debug-non-zts-20100525/pdo.so，这里需要有具体的文件路径，也就是上面编译完成后给出的路径
      • 最后重启相应的web服务或php-fpm即可
